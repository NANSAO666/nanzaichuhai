# 南崽出海 · 广州游艇AI运营系统

> 南崽出海，赴一场江海浪漫

一套纯前端的广州游艇运营双页面系统，无需后端，开箱即用。

## 项目介绍

本项目包含两份独立的单HTML文件：

| 文件 | 说明 |
|------|------|
| [管理员后台.html](./管理员后台.html) | 商家管理端：品牌配置、码头管理、图片素材、AI配置、订单台账等 |
| [客户前端.html](./客户前端.html) | 用户展示端：码头浏览、AI问答、智能报价、在线预约、订单查询 |

## 核心功能

### 管理员后台
- 🏪 **品牌商家配置** - 品牌名称、标语、联系方式、营业时间等
- 🛥️ **广州游艇俱乐部管理** - 五大码头完整信息（南沙、太古仓、长洲岛、莲花山、琶洲湾）
- 🖼️ **图片素材管理** - 多码头图片上传/压缩/设封面，Base64本地存储
- 🤖 **AI接口全局配置** - 兼容OpenAI协议，配置一次前后端共用
- 🔔 **企业微信推送** - Webhook配置，预约订单自动推送
- 📖 **说明书可视化编辑** - 客户端使用说明在线编辑
- 📋 **全量订单台账** - 订单列表、状态管理
- 🔄 **一键同步前端** - 数据版本号管理，LocalStorage分层隔离

### 客户前端
- 🏠 **品牌首页** - Hero大图、品牌简介、精选码头
- 🏝️ **码头筛选** - 按区域筛选、详情轮播、设施展示
- 💬 **AI智能问答** - 游艇出海专属AI顾问，7×24在线解答
- 🧑‍💼 **客户服务** - 一键转人工，微信/电话快速联系
- 💰 **智能报价计算器** - 船型选择、增值服务、超时加价测算
- 📝 **在线预约表单** - 完整校验，订单写入本地
- ❓ **使用说明书** - 内置图文说明弹窗
- 📱 **我的预约** - 手机号查询历史订单

## 技术栈

- **纯HTML + 原生JS** - 无需构建工具，双击即运行
- **Tailwind CSS** - CDN引入，响应式设计
- **LocalStorage** - 模拟服务端持久化存储，分层隔离
- **Canvas压缩** - 图片自动压缩为Base64
- **OpenAI兼容协议** - 支持各类免费/付费大模型

## 快速开始

### 方式一：直接打开
双击 `客户前端.html` 或 `管理员后台.html` 即可在浏览器中运行。

### 方式二：本地服务（推荐）
```bash
# Python 3
python3 -m http.server 8000

# 或 Node.js
npx serve .
```

访问：
- 管理端：http://localhost:8000/管理员后台.html
- 客户端：http://localhost:8000/客户前端.html

## 数据同步规则

采用 LocalStorage 分层隔离存储：

| 键名 | 说明 |
|------|------|
| `nzch_brand` | 品牌商家配置 |
| `nzch_clubs` | 游艇俱乐部数据 |
| `nzch_ai_config` | AI接口配置 |
| `nzch_webhook` | 企业微信Webhook |
| `nzch_manual` | 使用说明书 |
| `nzch_orders` | 预约订单台账 |
| `nzch_version` | 数据版本号 |

**同步流程**：管理员后台修改 → 点击「保存并同步」 → 版本号递增 → 客户前端刷新自动读取最新数据

## 部署到 GitHub Pages

1. Fork / 上传本项目到 GitHub 仓库
2. 进入仓库 Settings → Pages
3. Source 选择 `Deploy from a branch`
4. Branch 选择 `main` / `master`，目录选择 `/ (root)`
5. 保存后等待部署完成，即可公开访问

## 二次开发

代码采用模块化分层设计，便于后期拆分：

```
StorageModule    本地存储层
ImageModule      图片处理模块
SyncModule       数据同步模块
AIModule         AI请求模块
QuoteModule      报价计算模块
BookingModule    预约订单模块
ClubsModule      码头展示模块
ManualModule     说明书模块
UI               通用工具层
```

后期可直接拆分打包为 Uniapp / 手机 APP。
