# 南崽出海 · 云端连接部署指南

## 问题说明

之前管理员后台显示「本地模式」、订单无法同步的原因：
- 代码中使用的 `https://api.next.bspapp.com/database` 不是有效的uniCloud数据库API
- uniCloud的clientDB HTTP API不支持纯HTML页面直接调用
- 需要通过「云函数URL化」的方式来连接数据库

## 部署步骤

### 第一步：创建云函数

1. 登录 [uniCloud控制台](https://unicloud.dcloud.net.cn/)
2. 进入你的服务空间（支付宝云：mp-be8bb2ea-2253-4636-aef9-ad4968f829d4）
3. 点击左侧菜单「云函数」
4. 点击「新建云函数」
   - 云函数名称：`api`（或其他你喜欢的名字）
   - 点击确定

### 第二步：上传云函数代码

1. 在云函数列表中，找到刚创建的 `api` 函数
2. 点击「详情」
3. 在「函数代码」标签页中，将 `cloud-function-api/index.js` 的代码全部复制粘贴进去
4. 点击「保存」
5. 点击「部署」或「上传」（等待部署完成）

### 第三步：配置云函数URL化

1. 在云函数详情页，点击「URL化」或「HTTP访问」标签
2. 开启「启用HTTP访问」开关
3. 设置访问路径：`/api`（或其他路径）
4. 点击「保存」
5. 复制生成的完整HTTP地址，例如：
   - 阿里云：`https://xxx.api.unespress.cn/api`
   - 支付宝云：`https://xxx.api-hz.cloudbasefunction.cn/api`
   - 腾讯云：`https://xxx.service.tcloudbase.com/api`

### 第四步：配置管理员后台

1. 打开 `管理员后台.html`
2. 点击订单页面的「⚙ 配置云端」按钮
3. 将刚才复制的云函数HTTP地址粘贴到输入框
4. 点击「保存并连接」
5. 提示「云端连接成功」即配置完成

### 第五步：配置客户前端

有两种配置方式：

**方式一：URL参数传递（推荐）**
在客户前端链接后加上 `?funcUrl=你的云函数地址`，例如：
```
客户前端.html?funcUrl=https://xxx.api-hz.cloudbasefunction.cn/api
```

**方式二：在代码中修改默认配置**
修改 `客户前端.html` 中 CloudModule 的默认配置。

## 数据库表说明

确保你的uniCloud服务空间中有以下数据库表（集合）：

### nzch_config - 配置表
存储品牌、俱乐部、AI、webhook等配置信息

### nzch_orders - 订单表
存储客户预约订单信息

**注意：** 首次使用时，表会自动创建（如果云函数有创建表的权限）。如果无法自动创建，请手动在uniCloud控制台创建这两个表。

## 权限配置建议

为了安全，建议在云函数中添加简单的鉴权，例如：
- 在请求头中添加自定义token验证
- 或者限制只能从特定域名访问

## 测试验证

配置完成后：
1. 在客户前端提交一个预约订单
2. 在管理员后台点击「🔄 刷新订单」
3. 如果能看到刚提交的订单，说明同步成功！
