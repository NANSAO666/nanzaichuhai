## 一、系统概览

| 文件 | 用途 | 使用者 |

| 管理员后台.html | 品牌配置、俱乐部管理、图片素材、AI接口、Webhook、说明书编辑、订单台账 | 商家管理员 |

### 2.3 异常提示对照

| 提示 | 原因 | 解决 |
|------|------|------|
| 密钥失效(401) | API Key错误或过期 | 重新生成密钥 |

## 三、企业微信机器人Webhook获取步骤

### 3.1 创建企业微信群机器人

1. 打开「企业微信」App 或电脑端
2. 进入一个企业微信群(或新建一个商家通知群)
3. 点击群右上角「...」→「群机器人」→「添加机器人」
4. 命名机器人(如「南崽出海预约通知」)→ 添加
5. 复制生成的 **Webhook地址**,格式为:
   ```
   https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxxxxx-xxxx-xxxx
   ```

### 3.2 后台配置

1. 打开 `管理员后台.html`
2. 进入「微信推送」模块
3. 粘贴 Webhook 地址
4. 点击「发送测试消息」,企业微信群收到测试消息即成功
5. 点击「保存」

### 3.3 推送效果

客户在前端提交预约后,企业微信群自动收到完整订单通知(含订单编号、码头、客户、船型、时段、人数、增值配套、备注、商家对接方式),格式见后台「推送模板预览」。

> 注:Webhook推送需要联网;若推送失败不影响订单保存,订单仍写入台账。

## 四、后台码头图片上传替换操作流程

### 4.1 上传图片

1. 打开 `管理员后台.html`
2. 进入「图片素材」模块
3. 顶部下拉选择要管理的码头(如南沙游艇会)
4. 选择图片分类(码头实景/游艇实拍/航线风景/内饰图/宣传海报)
5. 点击「上传图片」按钮,可多选本地图片
6. 系统自动压缩为 Base64 存入本地(压缩至宽1280px、质量82%)
7. 上传完成自动刷新图片墙

### 4.2 设为封面

- 鼠标悬浮图片 → 点击「设封面」
- 封面图显示金色边框,前端首页与码头卡片优先展示封面
- 首张上传的图片自动设为封面
- 删除封面图后,自动将剩余首张设为封面

### 4.3 删除图片

- 鼠标悬浮图片 → 点击「删除」→ 确认

### 4.4 同步前端

- 任意修改后,点击顶部「一键同步前端」
- 客户前端刷新即可看到最新图片轮播
- 前端无图时自动展示默认精致游艇占位图标

### 4.5 存储说明

- 图片以 Base64 存入 LocalStorage,无需服务器
- LocalStorage 总容量约 5-10MB,建议单码头图片不超过 20 张
- 若存储空间不足,系统会弹窗提示精简图片

## 五、其他后台功能操作

### 5.1 品牌配置
修改品牌名、客服微信、电话、地址、营业时间、超时加价、品牌简介、品牌宣传大图。所有页面底部联系条、预约弹窗、AI问答、微信推送模板自动读取此配置。

### 5.2 俱乐部管理
- 新增码头:点击「新增码头」生成空白模板,在弹窗中填写完整信息
- 编辑码头:点击「编辑」修改名称、区域、地址、航线、设施、船型定价、增值服务、退款规则
- 停用码头:点击「停用」,前端自动隐藏(历史订单保留可查)
- 删除码头:在编辑弹窗底部「删除」(谨慎,历史订单保留但码头数据清除)

### 5.3 说明书编辑
- 新增章节、编辑标题与正文(支持换行)
- 上传配套操作示意图
- ↑↓ 按钮调整章节顺序
- 保存后同步前端「使用帮助」弹窗

### 5.4 订单台账
- 按码头/状态筛选订单
- 查看订单详情
- 切换订单状态(待确认/已成交/已取消)
- 导出 JSON 备份

### 5.5 系统重置
- 清空订单:仅清除订单,保留其他配置
- 重置俱乐部:恢复五大码头默认数据(自定义图片丢失)
- 全部清空:输入「我确认清空」后清空全部数据并恢复默认

## 七、后期升级APP改造适配方案

系统代码模块化分层清晰,预留 Uniapp/原生APP 改造接口:

### 7.1 模块平移对照

| Web 现状 | Uniapp 改造 |
|----------|-------------|
| `localStorage.getItem/setItem` | `uni.getStorageSync/setStorageSync` |
| `fetch()` AI请求 | `uni.request()` |
| `fetch()` Webhook推送 | `uni.request()` |
| `<input type="file">` 图片上传 | `uni.chooseImage()` + `uni.uploadFile` |
| Canvas 压缩 Base64 | `uni.compressImage()` |
| `<img>` + onerror 容错 | `<image>` 组件 + `@error` 事件 |
| 自定义轮播 | `<swiper>` 组件 |
| 底部 Tab 导航 | `pages.json` 配置 tabBar |
| SVG 图标 | 替换为 iconfont 字体图标 |
| LocalStorage 5MB 限制 | 改用 `uni.setStorage` 或云端存储 |

### 7.2 JS 模块复用

以下模块可 **直接平移** 至 Uniapp,无需重写:

- `StorageModule` — 仅替换 API 名称
- `AIModule.callAI()` — 改 `fetch` 为 `uni.request`
- `QuoteModule.calcQuote` — 纯计算逻辑完全复用
- `BookingModule.genOrderId` — 完全复用
- `BookingModule.pushWebhook` — 改 `fetch` 为 `uni.request`
- `DefaultDataModule` — 完全复用

### 7.3 拆分建议

将单 HTML 拆分为 Uniapp 多页面:

```
pages/
├── home/home.vue          ← HomeModule
├── clubs/list.vue         ← ClubsModule.render
├── clubs/detail.vue       ← ClubsModule.openDetail
├── ai/chat.vue            ← AIModule
├── ai/generator.vue       ← AIGenModule
├── booking/form.vue       ← BookingModule
├── booking/quote.vue      ← QuoteModule
├── mine/orders.vue        ← MineModule
└── manual/viewer.vue      ← ManualModule
```

### 7.4 后端升级路径

若后期需要多商家/多用户云端同步:

1. 后端采用 Node.js + Express + MySQL/PostgreSQL
2. 数据模型直接复用本系统 JSON 结构(见技术架构文档)
3. AI 请求改为后端代理(隐藏密钥)
4. Webhook 推送改为后端触发
5. 图片存储改用 OSS/COS 替代 Base64
6. 用户系统加入 JWT 鉴权

## 八、常见问题

**Q: 后台修改后前端没更新?**
A: 点击顶部「一键同步前端」,然后前端页面下拉刷新或重新打开。

**Q: 图片上传失败?**
A: 检查图片大小(建议单张<2MB),LocalStorage 容量约5MB,过多图片会撑爆。

**Q: AI 不回复?**
A: 检查后台 AI 接口配置是否填写密钥,点击「测试AI连接」验证。

**Q: 预约提交后商家没收到微信通知?**
A: 检查后台 Webhook 链接是否正确,点击「发送测试消息」验证。订单无论推送是否成功都会存入台账。

**Q: 换台电脑数据会丢吗?**
A: LocalStorage 仅存于当前浏览器。如需迁移,在后台「订单台账」点击「导出JSON备份」。后期升级APP后可云端同步。
